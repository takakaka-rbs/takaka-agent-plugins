# requirements-specificater

ユーザーとの対話・コードベース調査・Web調査を通じて、要件定義書テンプレートに沿った要件定義書を作成・更新するプラグインです。

## 概要

ユーザーが要件メモやドラフトを書くと、`/requirements-specificater` スキルが以下を行います。

1. 内容を要件定義書テンプレートの各セクションに整理する
2. 不足している情報を「コードベース調査」「Web調査」「ユーザーへの質問」で埋める
3. 機能要件・非機能要件・画面・外部インターフェースなどにIDを採番する
4. テンプレートに沿った要件定義書を完成させる

完成した要件定義書は、後続の **画面仕様書プラグイン** や **API仕様書プラグイン（OpenSpec含む）** から参照され、詳細仕様策定のインプットになります。

## 構成要素

| コンポーネント | ファイル | 説明 |
|---|---|---|
| Skill | `skills/requirements-specificater/SKILL.md` | `/requirements-specificater` スラッシュコマンド。対話の全体フローを制御する |
| Agent | `agents/requirements-researcher.md` | コードベース調査・Web調査を担当するサブエージェント |
| テンプレート | `docs/requirements-specification-template.md` | 要件定義書のベーステンプレート（ID採番ルール・記載ガイド付き） |

## ID採番ルール（後続プラグインとの連携）

要件定義書テンプレートでは、後続の画面仕様書・API仕様書プラグインからの参照を可能にするため、
以下のプレフィックス付きIDを使用します。

| プレフィックス | 対象 | 主な利用元 |
|---|---|---|
| `REQ-F-XXX` | 機能要件 | 画面仕様書・API仕様書プラグイン全般から参照 |
| `REQ-NF-XXX` | 非機能要件（性能・信頼性・セキュリティなど） | 各仕様書プラグインの非機能観点の前提として参照 |
| `SCR-XXX` | 画面 | **画面仕様書プラグイン**が本書のSCR-XXXをキーに詳細レイアウト・遷移を作成 |
| `IF-XXX` | 外部インターフェース / API | **API仕様書プラグイン（OpenSpec含む）**が本書のIF-XXXをキーにエンドポイント定義を作成 |
| `UC-XXX` | ユースケース | 機能要件の背景情報として参照 |
| `DAT-XXX` | データエンティティ | データモデル・APIスキーマ作成時の入力として参照 |
| `RISK-XXX` | リスク | プロジェクト管理・レビュー時に参照 |
| `OPEN-XXX` | 未確定事項（オープンイシュー） | 要件定義フェーズの残課題管理。後続プラグインの作業前に解消推奨 |

`XXX` は `001` から始まる3桁の連番です。**既存の要件定義書を更新する際、既存IDは変更しません**
（後続プラグインからの参照が壊れるため）。新規項目には新しい連番を割り当てます。

## 使い方

```
/requirements-specificater
```

新規作成・既存ファイルの更新・出力先パスについて対話形式で確認しながら進みます。

既存の要件定義書を更新する場合はパスを引数に渡せます。

```
/requirements-specificater docs/requirements/my-project.md
```

要件メモをそのまま渡して開始することもできます。

```
/requirements-specificater 在庫管理システムのリプレイス。既存システムはExcel運用。
```

## 今後の連携予定

このプラグインが出力する要件定義書をインプットに、以下のプラグインを作成予定です。

- **画面仕様書プラグイン**: 本書の `SCR-XXX` を画面一覧として、画面レイアウト・入力項目・遷移を詳細化
- **API仕様書プラグイン（OpenSpec含む）**: 本書の `IF-XXX` をAPI一覧として、エンドポイント・スキーマ・OpenAPI定義を詳細化

そのため、本プラグインの章構成・ID採番ルールは変更時に互換性に注意してください。

## インストール

```bash
PLUGIN=requirements-specificater
PROJECT=/path/to/your/project

cp -r agents/* $PROJECT/.claude/agents/
cp -r skills/* $PROJECT/.claude/skills/
```
