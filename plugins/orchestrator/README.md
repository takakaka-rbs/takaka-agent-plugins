# orchestrator

takaka-agent-plugins エコシステムの中枢プラグインです。SessionStart フックがルーティング表をメイン会話のコンテキストに注入し、**メイン会話自身がオーケストレーター**としてユーザーの依頼を適切な専門プラグインへ振り分けます。

## 概要

このプラグインは 1 つのフックと 1 つのエージェントと 1 つのスキルで構成されます。

| コンポーネント | 名前 | 役割 |
|---|---|---|
| Hook | `hooks/hooks.json`（SessionStart） | セッション開始時に `hooks/routing-table.md` のルーティング表をコンテキスト注入し、メイン会話をオーケストレーター化する |
| Agent | `orchestrator-planner` | 複雑タスクの分解。実行プランをJSON形式で出力 |
| Skill | `/route` | 特定プラグインへの直接ルーティング（自動ルーティングを迂回） |

## アーキテクチャ

```
SessionStart フック
      │  routing-table.md をコンテキスト注入
      ▼
User Request ──► メイン会話（＝オーケストレーター）
      │
      ├─ 実行前確認を伴う作業 ──► 該当スキルを Skill ツールで起動
      │                        （/requirements-specificater など）
      │
      ├─ 調査のみの独立作業 ──► *-researcher サブエージェントへ委譲
      │
      └─ 複雑なマルチステップタスク ──► orchestrator-planner
                                          │  (実行プランJSON を返す)
                                          ▼
                                step 1 → スキル or サブエージェント
                                step 2 → スキル or サブエージェント (step 1 完了後)
                                          │
                                          ▼
                                    結果を統合して返答
```

### 設計方針: なぜサブエージェントを経由しないのか

以前の設計では `orchestrator-master` サブエージェントが全リクエストのエントリーポイントでしたが、以下の理由で廃止しました。

- エージェントにはスラッシュコマンドが生成されず、ユーザーが確実に起動する導線がない
- サブエージェントはユーザーと対話できないため、対話形式のスキル（specificater 系）へ委譲できない
- サブエージェントからスキル（スラッシュコマンド）を起動できない

代わりに SessionStart フックでルーティング表を注入し、Skill ツールと Agent ツールの両方を使えるメイン会話がルーティングを担います。

## ルーティング規則（注入される内容の要約）

1. 依頼が担当タスク表に該当し**実行前確認（初回一括質問）を伴う**作業 → 該当スキルをメイン会話で起動して委譲
2. 依頼が担当タスク表に該当し**対話が不要な調査のみ**の独立作業 → 対応する `*-researcher` サブエージェントへ委譲
3. 複数プラグインにまたがる複雑なマルチステップタスク → `orchestrator-planner` に分解させてから各ステップを順次実行
4. どれにも該当しない依頼 → 通常どおり対応（無理にルーティングしない）

さらに**運用規則**として「質問はタスク開始時の最大4問・1回のみ（自明な項目は省略）」「作業開始後は質問禁止・報告のみで完遂」「曖昧な点は合理的な仮定で進めて最終報告に明記」「不可逆な破壊的操作（force push・ファイル削除・機密情報らしきファイルのコミット等）のみ実行前に必ず確認」を注入します。

全文は [`hooks/routing-table.md`](hooks/routing-table.md) を参照してください。

## 使い方

### 自動ルーティング（推奨）

プラグインをインストールして新しいセッションを開始し、通常通りチャットに依頼を書くだけです。ルーティング表が自動注入されているため、メイン会話が適切なスキル/エージェントへ振り分けます。

```
要件定義書を作りたい
→ メイン会話が /requirements-specificater スキルを起動し、実行前確認（1回）ののち自動で作成
```

### 直接ルーティング

特定のエージェントを指定したいときは `/route` スキルを使います。

```
/route requirements-specificater requirements-researcher タスクの説明
```

## 新しいプラグインを追加したとき

`hooks/routing-table.md` の **担当タスク一覧** テーブルに新しいプラグインのスキル/エージェントを追記してください。

```markdown
| 新プラグインが担当するタスクの説明 | `new-plugin-skill` | `new-researcher` |
```

## インストール

マーケットプレイス経由でインストールしてください。

```
/plugin marketplace add <takaka-agent-plugins のリポジトリ>
/plugin install orchestrator@takaka-agent-plugins
```

フック変更はセッション再起動後に反映されます。
