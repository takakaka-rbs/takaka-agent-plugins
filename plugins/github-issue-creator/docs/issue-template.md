# 既定Issueテンプレート

対象リポジトリに `.github/ISSUE_TEMPLATE/` が存在しない場合に使う、Issue本文の既定構成です。
Issue候補の種類（バグ / 機能追加 / タスク）に応じて使い分けてください。
不要なセクションは削除して構いません（空のセクションを残さない）。

---

## バグ報告

```markdown
## 概要

<!-- 何が起きているかを1〜3行で -->

## 再現手順

1.
2.
3.

## 期待する動作

## 実際の動作

<!-- エラーメッセージ・ログがあれば貼る（機密情報は除去） -->

## 関連コード

<!-- issue-drafterの調査結果から。例: `src/auth/session.ts:42` -->

## 環境・補足

<!-- OS、バージョン、発生頻度、確度が「推測」の情報はここに明記 -->
```

---

## 機能追加・改善

```markdown
## 背景・課題

<!-- なぜこの機能が必要か。関連する要件ID（REQ-F-XXXなど）があれば記載 -->

## 提案内容

<!-- 何をどう変えるか -->

## 受け入れ条件

- [ ]
- [ ]

## 関連コード・影響範囲

<!-- issue-drafterの調査結果から -->

## 補足

```
