# postgres-dev

PostgreSQL で構築された対象リポジトリの DB に対して、DDL（テーブル定義・変更）・DML（シード/サンプルデータ・データ移行）を**マイグレーション運用に沿って**作成・変更するプラグインです。

対象リポジトリのスキーマ管理方式（マイグレーションツール・ファイル命名規約・適用コマンド・コード生成）は実行時に調査して従うため、特定のリポジトリに依存しません。

## 概要

`/postgres-dev` スキルが以下を行います。

1. **実行前確認（唯一の質問タイミング）**: 対象リポジトリ・変更内容と入力文書・適用まで行うか・DML の扱いを、最大4問まで最初に1回だけ確認する（文脈から自明な項目は聞かない）
2. **スキーマ調査**: `postgres-researcher` がマイグレーション運用（ツール・連番規約・適用コマンド）・既存スキーマと命名規約・コード生成（jOOQ 等）・DB環境を調査する
3. **設計・報告**: テーブル定義（型・制約・インデックス）を既存スキーマの慣習と PostgreSQL の流儀に沿って設計し、報告して続行する（承認は待たない）
4. **マイグレーション作成**: **適用済みファイルは編集せず**、命名規約に従った次の連番で新規マイグレーションを作成する。データを壊さない手順（デフォルト値・段階的変更）とロールバック手順を含める
5. **適用・検証**: ローカル/テスト用DBへの適用、スキーマ由来コードの再生成（jOOQ codegen 等）、ビルド確認まで行う（実行前確認で選択した場合）
6. **完了報告**: 作成ファイル・スキーマ変更の要約・ロールバック手順・採用した仮定・未解決課題を開示する

**破壊的操作（DROP TABLE / DROP COLUMN / TRUNCATE / データを失う変更・適用済みマイグレーションの変更・ローカル以外への適用）だけは、実行前に必ず確認します。**

## 構成要素

| コンポーネント | ファイル | 説明 |
|---|---|---|
| Skill | `skills/postgres-dev/SKILL.md` | `/postgres-dev` スラッシュコマンド。実行前確認から適用・報告までの全体フローを制御する |
| Agent | `agents/postgres-researcher.md` | スキーマ管理方式・既存スキーマ・命名規約・コード生成・DB環境の読み取り調査 |

## 他プラグインとの関係

```
要件定義書 / OpenSpec.yml / 実装計画 … テーブル設計の根拠（REQ-F-XXX / API-XXX を記録）
  ↓ 参照
DDL / DML マイグレーション（本プラグイン）
  ↓ スキーマ由来コードの再生成（jOOQ 等）
アプリコードの実装 … /code-implementer の担当（本プラグインはアプリコードを修正しない）
```

- アプリコードの実装・修正が伴う場合は `/code-implementer` を使います（本プラグインは DB 側と生成コードの再生成まで）
- 作成したマイグレーションのコミット・プッシュは `/git-commit-pusher` に委ねます

## 使い方

```
/postgres-dev
```

引数で対象と内容を直接指定することもできます。

```
/postgres-dev users テーブルにメールアドレス確認済みフラグを追加したい
/postgres-dev /path/to/repo 商品と在庫のテーブルを新規設計（docs/requirements/inventory.md が根拠）
/postgres-dev 開発用のサンプルデータを追加したい
```

## 前提条件

- 対象リポジトリに PostgreSQL のスキーマ管理（Flyway 等のマイグレーションツール）が存在すること（存在しない場合は導入の選択肢を提案し、素の SQL 作成までに留めます）
- 適用まで行う場合、ローカル/テスト用DBが起動できること（docker-compose / devcontainer 等）

## インストール

```bash
# マーケットプレイス未登録の場合のみ
claude plugin marketplace add /path/to/takaka-agent-plugins

claude plugin install postgres-dev@takaka-agent-plugins
```
